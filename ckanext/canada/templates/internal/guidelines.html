The following criteria have been established for preparing datasets for publication to the Government of Canada’s Open Data Portal.

  1. The dataset must be available in the public domain.
  2. The provided link (URL) is a direct link to the dataset. Avoid providing links to a Web page that requires further navigation in order to find the dataset.
  3. The dataset is in structured, machine-readable and platform independent formats such as CSV/TXT, GeoTIFF, KML/KMZ, RDF, RSS/GeoRSS, SHP, XLS, or XML. Avoid PDF and HTML (eg. HTML tables).
  4. A file containing definitions of the headings (row and column) and variables used in the dataset(s) should be uploaded to the Portal, in XML or TXT format.
  5. The dataset should be subject to as little interpretation, analysis, and massaging as possible.
  6. Metadata must be provided in both official languages. Where a dataset is available in only English or French, enter the same URL for both where required and provide metadata for the dataset in both languages.
  7. The quality of the dataset is acceptable and is the responsibility of the department or agency.
  8. Compliance of the dataset with GC privacy and security policy and legislation is verified by the department or agency.

Publishers must also consider the potential of copyright and liability issues around the publication of datasets to the Open Data Portal.

Les critères suivants ont été établis pour les jeux de données qui doivent être publiés dans le Portail de données ouvertes du gouvernement du Canada.

  1. Le jeu de données doit être accessible dans le domaine public.
  2. Le lien fourni (URL) est un lien direct vers le jeu de données. Évitez de fournir des liens vers une page Web où l’utilisateur devra continuer de naviguer pour trouver le jeu de données.
  3. Le jeu de données est sous forme structurée, en formats lisibles par machine, indépendants, tels que : CSV/TXT, GeoTIFF, KML/KMZ, RDF, RSS/GeoRSS, SHP, XLS ou SML. Évitez les formats PDF et HTML (p. ex., les tableaux HTML).
  4. Un fichier contenant les définitions des en-têtes (rangées et colonnes) et des variables utilisées dans le jeu de données devrait être téléchargé dans le Portail, soit en format XML ou TXT.
  5. Le jeu de données devrait laisser le moins de place possible à toute interprétation, analyse ou traitement.
  6. Les métadonnées doivent être fournies dans les deux langues officielles. Lorsqu’un jeu de données n’est accessible que dans une langue, entrez la même adresse URL pour les deux versions et fournissez les métadonnées pour le jeu de données dans les deux langues.
  7. La qualité du jeu de données est acceptable et relève du ministère ou de l’organisme.
  8. La conformité du jeu de données aux politiques et aux lois du GC en matière de protection des renseignements personnels et de sécurité doit être assurée par le ministère ou l’organisme en question.

Les éditeurs doivent également tenir compte des questions possibles liées aux droits d’auteur et à la responsabilité, en ce qui a trait à la publication de jeux de données dans le Portail de données ouvertes.

Guide de mise en œuvre des métadonnées dans le cadre du Portail de données ouvertes
